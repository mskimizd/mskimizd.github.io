搭建Spring Boot开发环境

通常有以下几种方法搭建Spring Boot开发环境

  1. 直接下载 Spring Tool Suite 集成环境（包含Eclipse）安装使用
  2. 在现有的Eclipse基础上安装Spring Tool Suite插件
  3. 在Visual Studio Code上安装JAVA和Spring插件（STS4开始）

由于已经长期在用Eclipse，所以选择了安装插件的方式实现对Spring Boot的支持。

插件安装 在线 OR 离线?

Eclipse上Spring Tool Suite插件的安装可以分为在线与离线两种方式。

在线方式比较简单，通过Eclipse Marketplace中搜索Spring安装即可。缺点就是网速慢的话下载安装要耗费不少时间。此外，有一定几率安装会失败，这个网上也有不少同样的情况，都是在线安装的。
离线方式需要先下载Spring Tool Suit包，通过Install New Software来安装，具体教程百度一下。这里需要注意，Spring官网上的Spring Tool Suit包不同版本对应不同的Eclipse版本。确保下载到相对应的版本才能正确安装。

这次我选择了在线安装，因为网络原因，下离线包的速度也是特别慢，不如直接在线安装。第一次安装出现了失败的情况。后面重新安装成功了。

创建一个Spring Boot Demo

安装完成之后，New Project中就出现了Spring Boot, 选择Spring Starter Project，后面可以不需要修改什么什么，直接下一步到完成即可，反正是demo。

这里需要注意的一点时，第一次创建时，通过Maven构建项目要花很久，同样是网络原因嘛，最好先将Maven的源更改为国内的源比如阿里的。

配置Maven源

选择某个目录，新建一个配置好源的setting.xml文件和repository文件夹
Window->preferences->Maven->User Settings
User Settings栏选择新建的setting.xml，点击open file，然后apple and close。
完成！

额外：关于kafka

在新建的demo工程配置了kafka和kafka_client，写了一个简单的生产者消费者demo。发现消费者启动时，kafka_client会打印一堆DEBUG日志，再三查阅资料未果，遂放弃。因为中间发现了spring-kafka这个东西，特别好用，在Spring Boodt中写这样一个demo更简单，也没有日志问题。具体详情后续博文再更~
